What is Cloud Computing?
Delivering the IT resources like Storage , Server and network through the Internet instead of using in Locally.
  
IaaS:
the cloud provider will provide the server, the user should maintain the OS , application and Coding.
IaaS = Renting a computer server online

PaaS:
The PaaS will provides the ready-to-use platform where you can develop and deploy applications without worrying about servers.

The cloud provider manages:
Hardware
Operating system
Runtime environment

You only focus on writing and deploying code.

PaaS = Platform ready for coding and deploying apps

SaaS:
The SaaS will provides the all developed software that users can access. use can the software without any installation.

SaaS = Software ready to use online 
  
| Service | What you manage       | Example           |
| ------- | --------------------- | ----------------- |
| IaaS    | Applications, OS      | AWS EC2           |
| PaaS    | Only application code | Google App Engine |
| SaaS    | Nothing (just use it) | Google Docs       |

wht is devops?
collaboration between development and operation team to deploy code to production in automated way.

devops life cycle
plan
code - git,jira,confluence
build - sbt , maven
test - Junit , Selenium
release - Jenkins , codeship
deploy - Docker , AWS
operate - Chef , Ansible , Kubernetes
monitor - Nagios , splunk , Datadog

whats in it for you?
collaboration and trust
release faster and work smarter
accelerate time to resolution 
better manage unplanned work

devops vs agile

1.Agility           | agility in both development and operations  | only development
2.process/practice  | involve process such as CI,CD,CT            | agile scrum , agile Kanban
3.focus area        | timeliness , quality have equal priority    | Timeliness is the main priority
4.scope of work     | agility and need for automation             | agility only
5.release cycle and | smaller release cycle                       | smaller release cycle
  development sprint| immediate feedback                          |

devops principles

automation , continuous delivery and fast reaction to feedback

3.Linux server admin experience or a deep understanding of Linux
4.hands-on experience using puppet or chef any other configuration management tool
